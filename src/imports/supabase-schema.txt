You are a senior backend engineer. Help me implement the Supabase backend for a Smart Energy Optimization Dashboard used by college campuses.

The frontend is already built using React + TypeScript (generated by Figma Make). I now need a proper Supabase database instead of placeholder data.

Your task is to design and implement the Supabase backend schema and provide SQL for table creation.

Project Requirements:

This system monitors campus energy usage, performs analytics, and provides AI-based predictions and optimization recommendations.

Create the following PostgreSQL tables in Supabase.

1. Table: campuses

Purpose: Store information about each campus.

Columns:

* id (uuid, primary key, default uuid_generate_v4())
* name (text, not null)
* location (text)
* num_buildings (integer)
* created_at (timestamp, default now())

2. Table: buildings

Purpose: Store buildings belonging to a campus.

Columns:

* id (uuid, primary key)
* campus_id (uuid, foreign key referencing campuses.id)
* building_name (text)
* building_type (text) (examples: academic, library, hostel, lab)
* floor_area (integer)
* created_at (timestamp default now())

3. Table: energy_data

Purpose: Store historical energy usage data.

Columns:

* id (uuid primary key)
* campus_id (uuid references campuses.id)
* building_id (uuid references buildings.id)
* timestamp (timestamp)
* occupancy (integer)
* temperature (numeric)
* humidity (numeric)
* energy_consumption (numeric) (kWh)

4. Table: predictions

Purpose: Store AI model predictions for future energy consumption.

Columns:

* id (uuid primary key)
* campus_id (uuid references campuses.id)
* prediction_time (timestamp)
* predicted_energy (numeric)
* efficiency_score (integer)
* recommendation (text)
* created_at (timestamp default now())

5. Table: user_inputs

Purpose: Store manual inputs entered by users in the dashboard.

Columns:

* id (uuid primary key)
* campus_id (uuid references campuses.id)
* input_date (date)
* input_time (time)
* occupancy (integer)
* temperature (numeric)
* created_at (timestamp default now())

Additional Requirements:

1. Provide the complete SQL schema for Supabase.
2. Add foreign key relationships.
3. Add indexes for timestamp and campus_id to optimize queries.
4. Ensure the schema supports future AI/ML predictions.
5. Provide example INSERT queries to populate demo data.
6. Suggest how the React frontend should query these tables using the Supabase JS client.

The goal is to replace all placeholder dashboard data with real data from Supabase.
